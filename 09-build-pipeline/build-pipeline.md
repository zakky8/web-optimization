# Build Pipeline

A well-structured asset pipeline is the difference between a 3D scene that loads in 2 seconds and one that loads in 15. This covers the tools that transform raw artist exports into optimized, CDN-ready assets.

## gltfjsx

gltfjsx converts `.glb`/`.gltf` files into declarative React Three Fiber (R3F) JSX components. It introspects the model structure and generates typed, reusable component code.

### Installation and Basic Use

```bash
# One-off (no install)
npx gltfjsx model.glb

# With options
npx gltfjsx model.glb -o Model.jsx --types --transform --draco
```

### Key Flags

| Flag | Effect |
|---|---|
| `--types` | Generate TypeScript `.tsx` output |
| `--draco` | Add Draco decoder path to loader |
| `--transform` | Run meshopt/draco compression on the model |
| `--shadows` | Add `castShadow receiveShadow` to all meshes |
| `--precision 2` | Reduce float precision in generated code |
| `--root /models` | Set public base path for model URL |

### Generated Output Example

```tsx
// auto-generated by gltfjsx — do not edit directly
import { useGLTF, useAnimations } from "@react-three/drei";
import { GLTF } from "three-stdlib";

type GLTFResult = GLTF & {
  nodes: {
    Body: THREE.Mesh;
    Wheel_FL: THREE.Mesh;
    Wheel_FR: THREE.Mesh;
  };
  materials: {
    CarPaint: THREE.MeshStandardMaterial;
    Rubber:   THREE.MeshStandardMaterial;
  };
};

export function Car(props: JSX.IntrinsicElements["group"]) {
  const { nodes, materials } = useGLTF("/models/car.glb") as GLTFResult;

  return (
    <group {...props} dispose={null}>
      <mesh geometry={nodes.Body.geometry}   material={materials.CarPaint} />
      <mesh geometry={nodes.Wheel_FL.geometry} material={materials.Rubber} />
      <mesh geometry={nodes.Wheel_FR.geometry} material={materials.Rubber} />
    </group>
  );
}

useGLTF.preload("/models/car.glb");
```

```ts
import { defineConfig }     from "vite";
import react                from "@vitejs/plugin-react";
import glsl                 from "vite-plugin-glsl"; // shader imports

  plugins: [
    react(),

    }),

  ],

  assetsInclude: [
    "**/*.glb",           // include binary assets in asset pipeline
    "**/*.gltf",
    "**/*.hdr",
    "**/*.ktx2",
  ],

  build: {
    target: "es2022",     // enables top-level await for WebGPU
    rollupOptions: {
      output: {
        manualChunks: {
          three:   ["three"],
          drei:    ["@react-three/drei"],
          rapier:  ["@dimforge/rapier3d-compat"],
        },
      },
    },
    assetsInlineLimit: 0, // never inline GLB as base64 — always external file
  },

  optimizeDeps: {
    exclude: ["@dimforge/rapier3d-compat"], // WASM — exclude from pre-bundling
  },

  server: {
    headers: {
      // Required for SharedArrayBuffer (Rapier worker, Atomics)
      "Cross-Origin-Opener-Policy":   "same-origin",
      "Cross-Origin-Embedder-Policy": "require-corp",
    },
  },
```


## Asset Optimization Pipeline

### GLTF/GLB Compression

```bash
# Install tools
npm install -g @gltf-transform/cli

# Inspect a model
gltf-transform inspect model.glb

# Full optimization pass
gltf-transform optimize model.glb model-opt.glb \
  --compress draco \          # Draco mesh compression
  --texture-compress webp \   # WebP textures
  --texture-size 1024 \       # Max texture dimension
  --simplify                   # Mesh simplification (LOD)

# Separate LODs
gltf-transform lod model.glb model-lod.glb \
  --lod-ratio 0.5 \
  --lod-ratio 0.25
```

### KTX2 Texture Compression

KTX2 with Basis Universal transcodes to the best GPU format at runtime (BCn, ASTC, ETC2):

```bash
# Install Basis encoder
npm install -g basisu

# Encode
basisu texture.png -output_file texture.ktx2 -uastc

# Three.js loading (requires KTX2Loader)
import { KTX2Loader } from "three/addons/loaders/KTX2Loader.js";

const ktx2Loader = new KTX2Loader()
  .setTranscoderPath("/basis/")
  .detectSupport(renderer);

const loader = new GLTFLoader();
loader.setKTX2Loader(ktx2Loader);
```

## Automated Asset Pipeline Script

```js
// scripts/optimize-assets.mjs
import { execSync } from "child_process";
import { readdirSync, statSync } from "fs";
import path from "path";

const INPUT_DIR  = "assets/raw";
const OUTPUT_DIR = "public/models";

const files = readdirSync(INPUT_DIR).filter(f => f.endsWith(".glb"));

for (const file of files) {
  const input  = path.join(INPUT_DIR, file);
  const output = path.join(OUTPUT_DIR, file.replace(".glb", "-opt.glb"));
  const before = statSync(input).size;

  execSync(
    `gltf-transform optimize "${input}" "${output}" ` +
    `--compress draco --texture-compress webp --texture-size 1024`,
    { stdio: "inherit" }
  );

  const after = statSync(output).size;
  const saving = (((before - after) / before) * 100).toFixed(1);
  console.log(`${file}: ${(before / 1024).toFixed(0)}KB → ${(after / 1024).toFixed(0)}KB (${saving}% smaller)`);
}
```

## CI/CD Integration

### GitHub Actions Workflow

```yaml
# .github/workflows/optimize-assets.yml
name: Optimize 3D Assets

on:
  push:
    paths:
      - "assets/raw/**"

jobs:
  optimize:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install tools
        run: |
          npm install -g @gltf-transform/cli
          npm ci

      - name: Optimize assets
        run: node scripts/optimize-assets.mjs

      - name: Generate JSX components
        run: |
          for f in public/models/*.glb; do
            npx gltfjsx "$f" --types --transform -o "src/models/$(basename ${f%.glb}).tsx"
          done

      - name: Commit optimized assets
        uses: stefanzweifel/git-auto-commit-action@v5
        with:
          commit_message: "chore: optimize 3D assets [skip ci]"
          file_pattern: "public/models/*.glb src/models/*.tsx"
```

### Pre-commit Hook

```bash
# .husky/pre-commit
#!/bin/sh
# Check for unoptimized GLBs > 2MB
for f in $(git diff --cached --name-only | grep ".glb$"); do
  size=$(stat -c%s "$f" 2>/dev/null || stat -f%z "$f")
  if [ "$size" -gt 2097152 ]; then
    echo "ERROR: $f is larger than 2MB. Run: gltf-transform optimize $f $f"
    exit 1
  fi
done
```

## Bundle Analysis and Targets

```bash
# After build, open bundle visualizer
npx vite build && open dist/stats.html
```

Target budgets for 3D scenes:

| Asset | Budget |
|---|---|
| Main JS bundle (Three.js + app) | < 350 KB gzip |
| Largest GLB model | < 1.5 MB |
| Total texture memory (GPU) | < 64 MB mobile / 256 MB desktop |
| Initial load (critical path) | < 500 KB |
| Time to first render | < 2.5 s on 4G |

## Performance Implications

- **Code splitting**: three.js (~600 KB minified) should be a separate chunk so returning users hit cache.
- **Preload models**: use `useGLTF.preload("/model.glb")` at module level so loading starts before the component mounts.
- **draco vs meshopt**: Draco gives better compression; meshopt decodes faster. Use Draco for download size, meshopt when CPU decode speed matters (mobile).
- **Texture atlasing**: fewer texture objects = fewer GPU state changes. Combine small textures into atlases.
- **Basis Universal transcoding**: let the GPU cache warm up by transcoding on idle, not on first visible frame.
